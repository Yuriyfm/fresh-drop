# Spotify Data Notes

## Назначение файла

Этот файл хранит рабочие заметки по Spotify API.

Не считай эти заметки вечной истиной. При реализации проверяй актуальную документацию Spotify API.

## Базовый подход MVP

Для поиска свежих релизов использовать Spotify Search API с запросом `tag:new`.

Примерная идея:

- искать альбомы/релизы через `tag:new`;
- фильтровать по дате релиза;
- получать детали альбома;
- получать данные артистов;
- обогащать релиз жанрами, popularity и market/country-данными.

## Важные ограничения

Spotify API может не давать прямую страну артиста.

Если прямой страны артиста нет, возможные fallback-варианты:

- market availability;
- внешние данные, если позже будут добавлены;
- значение `unknown`.

Для MVP не строить сложную систему определения страны, если данные ненадёжны.

## Жанры

Spotify обычно связывает genres с артистами, а не напрямую с релизами.

Для MVP жанры релиза можно получать как объединение жанров:

- главного артиста;
- всех артистов релиза.

Если genres отсутствуют, использовать пустой массив.

## Popularity

Popularity может быть доступна на уровне артиста, трека или альбома в зависимости от endpoint.

Для MVP фильтр популярности можно строить на наиболее стабильном доступном поле.

Важно явно определить в spec, что именно означает “популярный”:

- popularity главного артиста;
- средняя popularity артистов;
- popularity релиза, если доступна.

## Release date

Spotify release date может иметь разную точность:

- year;
- month;
- day.

Для фильтрации по последним 7/14/30 дням нужны даты с точностью до дня.

Если точность даты ниже дня, релиз нельзя надёжно включить в дневные фильтры.

Для MVP такие релизы можно:

- исключать из date-sensitive фильтра;
- или включать только в месячный режим, если это явно описано в spec.

## Что нужно проверить при реализации

- Возвращает ли `tag:new` только альбомы или также singles.
- Можно ли стабильно получить `single`, `album`, `compilation`.
- Какие лимиты API влияют на сбор данных.
- Как работает пагинация Search API.
- Насколько полно Search API отдаёт свежие релизы за 7/14/30 дней.
- Нужно ли кеширование результатов.
