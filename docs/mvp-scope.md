# MVP Scope

## Цель MVP

Проверить, нужен ли пользователям отдельный сервис для поиска свежих релизов Spotify по дате выхода и фильтрам.

## Входит в MVP

### Поиск свежих релизов

Периоды:

- 7 дней;
- 14 дней;
- 1 месяц.

### Источник данных

Использовать Spotify API.

Базовый подход:

- поиск релизов через `tag:new`;
- получение деталей релизов;
- получение данных артистов;
- обогащение релизов жанрами, страной и популярностью, если данные доступны.

### Фильтры

MVP должен поддерживать фильтры:

- период выхода;
- жанр;
- страна артиста;
- популярность;
- тип релиза.

### Типы релизов

Поддерживаемые типы:

- single;
- album;
- compilation, если доступно из API.

Если Spotify Search API через `tag:new` не позволяет удобно получать синглы, в MVP допустимо временно ограничиться альбомами, но это должно быть явно отражено в UI и документации.

### Жанры

Жанры берутся:

- у главного артиста;

Жанры релиза могут быть объединением жанров всех связанных артистов.

### Страна

Страна определяется по данным артиста или market-данным, если прямой страны артиста нет.

Если страна не может быть определена надёжно, значение должно быть `unknown`.

### Карточка релиза

Карточка релиза должна показывать:

- обложку;
- название релиза;
- артиста или артистов;
- тип релиза;
- дату выхода;
- жанры;
- страну или market;
- popularity, если доступно.

## Не входит в MVP

- Spotify login.
- Персональные рекомендации.
- Сохранение релизов в Spotify.
- Создание плейлистов.
- Лайки, закладки, личная библиотека.
- История просмотров.
- Социальные функции.
- Сложная рекомендательная система.
- Мобильные приложения.
- Админка.
- Платежи.
- Полнотекстовый поиск по всей истории Spotify.
- Хранение большой исторической базы релизов за годы.

## Хранение данных

MVP не должен хранить большой исторический архив.

Идеальная модель:

- хранить только свежие релизы за последний месяц;
- автоматически удалять устаревшие данные;
- периодически подгружать новые релизы.

## Важное ограничение

Если какое-то поле невозможно получить из Spotify API стабильно и недорого, не усложнять MVP.

Лучше явно показать ограничение, чем строить ненадёжную логику.
