# UX Summary

Краткая UX-памятка для разработки. Полная версия: `docs/specs/ux.spec.md`.

## Суть продукта

Mobile-first веб-приложение для поиска свежих релизов Spotify по фильтрам.

Главный сценарий:

```text
открыл приложение → выбрал период/жанр/страну → нашёл релиз → открыл страницу релиза → ушёл слушать в Spotify
```

Приложение не является рекомендательной системой. Оно показывает релизы по выбранным фильтрам.

## Главные UX-принципы

* Mobile-first: удобство на телефоне важнее десктопа.
* Search-first: это каталог/поисковик, а не рекомендательная лента.
* Главная единица интерфейса — релиз.
* Главная цель — быстро открыть подходящий релиз в Spotify.
* В списке показывать минимум данных, детали — на странице релиза.
* Визуальный стиль: близко к Spotify, dark theme по умолчанию, зелёный акцент.
* Не добавлять не-MVP функции без отдельного решения.

## Экраны MVP

1. Главная страница поиска.
2. Страница релиза: `/releases/:id`.
3. How it works overlay на search page.

## Главная страница

По умолчанию выбран период:

```text
Last 7 days
```

На главной странице:

* header;
* короткое описание продукта;
* overview за выбранный период;
* фильтры;
* summary выдачи;
* список релизов.

Фильтры доступны сразу. Overview и список во время загрузки показывают skeleton.

## Overview

Показывает общую картину свежих релизов за период:

* всего релизов;
* топ жанров;
* топ стран/markets;
* известные артисты с новыми релизами.

Не перегружать overview аналитикой.

## Фильтры MVP

* period: `7 days / 14 days / Month`;
* genres;
* release type: `album / single / compilation`;
* sorting.

Фильтры применяются сразу после выбора.

Выбранные фильтры и сортировка синхронизируются с URL query params, чтобы ссылкой можно было делиться. Local storage используется только как fallback-preferences, если в URL нет явного значения. `Reset filters` сбрасывает и текущий UI, и сохранённое состояние.

На мобильном:

* period и genre видны сверху;
* period использует segmented control `7 days / 14 days / Month`;
* mobile search controls визуально разделены на отдельные блоки `Period`, `Filters`, `Genre`, при этом compact trigger `Filters` находится под period и над genre;
* genre занимает всю ширину и в свернутом состоянии не вытесняет список релизов за первый экран;
* genre search занимает всю ширину, выбранные жанры показываются chips прямо внутри search field;
* если выбраны жанры, справа от search field доступно явное действие `Clear genres`;
* одновременно можно выбрать не больше трёх жанров;
* mobile filters и summary не должны раздуваться по вертикали: paddings и helper text остаются компактными, чтобы первый экран отдавал приоритет списку релизов;
* genre rows используют кастомный themed checkbox вместо системного blue checkbox;
* вся строка жанра остаётся единым touch target не ниже `44px`;
* список жанров имеет ограниченную высоту и внутренний scroll;
* если поиск жанров ничего не нашёл, показывается `No genres found`;
* count у жанра показывается компактным badge без слова `releases` в каждой строке;
* helper text под списком один раз сообщает, что genre count рассчитаны по окну `Month`, а не по текущему периоду;
* остальные фильтры в bottom sheet;
* type в bottom sheet не должен сжимать текст на узких экранах; допустим двухрядный segmented control;
* sorting открывается через compact trigger и bottom sheet, а не системный select;
* в bottom sheet также есть `Reset filters`;
* language находится в header/settings area и не конкурирует с фильтрами за главное внимание;
* desktop language switcher использует compact dropdown с полными self-name названиями языков, а не segmented toggle `EN/RU`.
* над списком релизов есть compact sticky summary bar с количеством результатов и human-readable контекстом активных фильтров;
* если mobile filter panel ещё виден на экране, summary не дублирует кнопку `Filters`;
* reset filters на мобильном доступен из summary как компактный text-link action, а не как вторая крупная CTA-кнопка.
* sticky top header на search page не нужен; после долгого скролла появляется компактная floating-кнопка возврата наверх.
* mobile settings показывают секцию `Language` с полными названиями языков, а не двухбуквенный переключатель.
* при закрытом genre list первый экран по возможности показывает не меньше трёх release cards без длинного скролла.

На десктопе (`>= 1024px`):

* страница поиска использует layout `filters sidebar + content`;
* sidebar с фильтрами находится слева, имеет фиксированную визуальную ширину и sticky-позиционирование;
* внутри sidebar фильтры складываются вертикально и занимают всю доступную ширину, а не пытаются повторять широкую двухколоночную desktop-сетку;
* `Genre search` в sidebar не должен сжиматься второй колонкой; `Type` может использовать компактную сетку `2x2`, если это нужно для узкой панели;
* summary и sorting справа собираются в компактный raised header-panel, чтобы правая колонка начиналась с той же визуальной линии и плотности, что и sidebar;
* sorting находится над списком справа, рядом с summary выдачи, а не внутри общей панели фильтров;
* reset filters находится внутри search sidebar как часть search-инструментов, а не в отдельном result-toolbar;
* header остаётся сверху, но становится компактнее и не отнимает большую вертикальную область;
* общая ширина контента ограничена, чтобы layout оставался аккуратным на `1024 / 1280 / 1440+`.

Ограничения MVP:

* можно выбрать несколько жанров, но не больше трёх одновременно;
* в списке жанров есть вариант `No genre` / `Без жанра` для релизов без жанров Spotify metadata;
* общие верхнеуровневые жанры показываются выше узких жанров, если по ним есть релизы;
* sorting default: `Newest first`;
* popularity is a sorting mode, not a filter.

## Список релизов

Результаты показываются списком, не сеткой.

Строка релиза:

* cover;
* title;
* artist;
* compact genres preview from `release.genres`;
* release date;
* release type badge;
* chevron/индикатор перехода.

Вся строка кликабельна и открывает страницу релиза.

На мобильном карточка релиза:

* имеет стабильную высоту примерно `80-88px`;
* использует обложку `64x64` или `72x72`;
* показывает placeholder/skeleton, пока cover ещё загружается;
* имеет более заметный title и читаемые вторичные metadata;
* жанры остаются в одну компактную строку и не раздувают карточку по высоте;
* имеет pressed state, а на десктопе hover/focus state.

Кнопки Spotify в списке нет.

Над списком показывать summary:

```text
128 releases found · Death Metal · Sweden · Last 7 days
```

Для списка используется automatic infinite scroll без кнопки `Load more`.

Чтобы не перегружать страницу при долгом скролле, UI виртуализирует список: загруженные релизы остаются в состоянии, но в DOM отрисовывается только видимый диапазон с запасом. При прокрутке вверх ранее загруженные релизы снова появляются.

Обязательные состояния:

* skeleton rows для первого load;
* при refetch старый список не исчезает мгновенно, а summary показывает компактный loading indicator;
* empty state с подсказкой следующего шага и быстрыми действиями (`Clear genres / Reset all filters / Use Month`);
* error state с human-readable сообщением и `Try again`, без технических деталей API;
* end state, визуально стоящий ниже списка, а не поверх него.

## Страница релиза

Страница релиза нужна, чтобы пользователь подтвердил выбор перед переходом в Spotify.

MVP route:

```text
/releases/:id
```

На странице:

* на мобильном верхняя панель релиза визуально сливается с краем экрана и не оставляет вокруг себя «рамку» страницы;
* верхняя панель релиза не дублирует основную CTA-кнопку `Open in Spotify`;
* верхняя панель релиза может ограничиться заметной back-icon без текстового дублирования названия релиза;
* обложка остаётся заметной, но на мобильном не занимает почти весь первый экран;
* mobile cover ограничен примерно `70vw / 320px`, а placeholder cover визуально компактнее и менее доминирует;
* название релиза;
* артист / артисты;
* заметная primary CTA `Open in Spotify`;
* secondary back-action не должен дублировать already-visible back control в верхней панели;
* metadata card c `release date / country / popularity / type`;
* на мобильном metadata card читается одной колонкой, построчно;
* type показывается только в metadata card, без дублирования рядом с hero;
* жанры главного артиста показываются chips, а не строкой;
* label, если есть;
* количество треков;
* свернутый tracklist, если доступен;
* на мобильном компоновка стремится уместить hero + title + artist + actions в первый экран без лишнего скролла.

Не показывать:

* жанры всех артистов;
* copyright;
* технические Spotify ID;
* raw API data.

При возврате назад желательно сохранять фильтры, позицию скролла и загруженный список.

## Spotify

`Open in Spotify`:

* на десктопе открывает Spotify в новой вкладке;
* на мобильном пытается открыть Spotify app;
* если приложение не установлено — открывает Spotify web.

## Theme

Dark theme по умолчанию.

Тёмная тема должна использовать явные design tokens для background / surface / raised surface / border / text / accent, чтобы карточки, панели, поля и фон читались как разные уровни одной системы, а не как почти одинаковые тёмные прямоугольники.

В header нужен переключатель темы:

* `Dark`;
* `Light`;
* опционально `System`, если легко.

Выбор темы сохраняется локально.

## i18n

Язык по умолчанию: English.

Сразу поддержать:

* English;
* Russian.

В header нужен переключатель языка.

Для search page header:

* на мобильном в header остаются `Fresh Drop`, короткий subtitle и компактные actions;
* `How it works` открывается без смены route: как centered modal на десктопе и как bottom sheet / full-screen sheet на мобильном;
* после закрытия `How it works` пользователь возвращается к тому же месту списка и с теми же активными фильтрами.
* `How it works` остаётся доступным, но визуально вторичен;
* language на мобильном открывается через settings/menu, а не занимает место в основном header;
* search page не добавляет отдельный sticky top bar поверх контента; быстрый возврат к началу решается floating-кнопкой.
* все sheet/modal используют общий close button с одинаковой геометрией и touch target.

Требования:

* переключение без перезагрузки;
* выбранный язык сохраняется локально;
* все UI-строки вынесены в i18n;
* не хардкодить тексты в компонентах.

## About / How it works

Страница объясняет:

* что делает приложение;
* что результаты основаны на фильтрах;
* что это не рекомендательная система;
* что часть данных может отсутствовать из-за ограничений Spotify metadata.

## Не входит в MVP

Не добавлять без отдельного решения:

* Spotify authorization;
* персональные рекомендации;
* избранное;
* отметку “прослушано”;
* скрытие релизов;
* sharing;
* PWA;
* отдельное мобильное приложение;
* audio preview/player;
* сложную иерархию жанров;
* выбор нескольких стран;
* автоприменение сохранённых фильтров при следующем открытии.
