# WSL Environment

## Назначение

Этот документ описывает особенности локального окружения WSL и правила работы с файлами для проекта.

Открывай этот файл только если задача касается установки зависимостей, запуска команд, dev-сервера, чтения или записи файлов, путей, прав доступа, кешей, временных файлов или проблем WSL/Windows.

## Рабочая директория

Проект находится в WSL:

```text
/home/yuriy/projects/fresh-drop
```

Все команды проекта выполняй из этой директории.

Не используй Windows-пути вида `C:\...` или `/mnt/c/...` для файлов проекта, если задача явно этого не требует.

## Чтение и запись файлов

Правила:

- читать можно файлы проекта и релевантные документы из `docs/`;
- писать можно только внутри проекта;
- не изменять файлы вне проекта без явного разрешения;
- не удалять пользовательские файлы без явной просьбы;
- не использовать destructive-команды вроде `git reset --hard` или массового удаления;
- для ручных правок файлов использовать точечные изменения, а не переписывать большие файлы целиком.

Если команда записи или удаления блокируется окружением, не обходи это через небезопасные команды. Используй безопасный patch-механизм или попроси пользователя о разрешении, если оно действительно нужно.

## Yarn

Проект использует Yarn 1.x.

Команды:

```text
yarn install
yarn dev
yarn test:run
yarn build
```

В WSL могут быть недоступны домашние cache/global директории Yarn. Для проекта зафиксированы локальные настройки в `.yarnrc`:

```text
cache-folder "./.yarn-cache"
global-folder "./.yarn-global"
```

Эти папки служебные и не должны попадать в git.

Если Yarn или пакетные postinstall-скрипты пытаются писать во временную директорию Windows и падают с ошибками прав доступа, запускай команды с WSL temp:

```text
TMPDIR=/tmp yarn install
TMPDIR=/tmp yarn test:run
TMPDIR=/tmp yarn build
```

## Dev-сервер

Для локального запуска используй:

```text
TMPDIR=/tmp yarn dev --host 0.0.0.0
```

Обычно Vite доступен по адресу:

```text
http://localhost:5173/
```

Если порт занят, Vite выберет другой порт. Используй URL из вывода команды.

## Генерируемые файлы

Не коммить и не редактируй вручную:

- `node_modules/`;
- `dist/`;
- `.yarn-cache/`;
- `.yarn-global/`;
- `*.tsbuildinfo`;
- локальные `.env` файлы.

Если сборка создала лишние служебные файлы в корне проекта, сначала проверь, почему они появились, затем исключи их через конфигурацию или `.gitignore`.

## Git

В этом окружении `.git` может быть доступен только частично или только на чтение.

Для проверки локального состояния проекта используй wrapper:

```text
./git-local status --short
./git-local diff
```

`git-local` запускает обычный `git` с `--git-dir=.git-local` и `--work-tree` на корень проекта. Это позволяет смотреть status/diff даже когда стандартная директория `.git` недоступна для обычных git-команд.

Если `./git-local` недоступен или команда падает, не считай это ошибкой проекта. Продолжай работу с файлами напрямую и укажи в отчёте, что git-статус не проверялся.

## Сеть

Сетевой доступ может быть ограничен.

Если установка пакетов или запрос к внешнему API не работает из-за сети:

- не менять код только ради обхода сетевой проблемы;
- зафиксировать точную команду, которую нужно повторить;
- продолжить локальные изменения, если это возможно без сети.
