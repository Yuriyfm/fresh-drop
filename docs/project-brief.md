# Project Brief

## Идея продукта

Приложение помогает искать свежую музыку, добавленную в Spotify за недавний период.

Фокус не на персональных рекомендациях “новой музыки для пользователя”, а на объективно новых релизах по дате выхода.

Пользователь может искать релизы за:

- последние 7 дней;
- последние 14 дней;
- последний месяц.

## Базовая концепция

Пользователь открывает приложение и видит свежие релизы Spotify.

Он может фильтровать их по:

- жанрам;
- странам;
- популярности артистов;
- типу релиза: single / album / compilation, если эти данные доступны.

## Spotify-авторизация

Для MVP логин через Spotify не требуется.

MVP работает без пользовательской авторизации Spotify.

На будущее можно заложить возможность гибридного режима:

- без авторизации — общий поиск свежих релизов;
- с авторизацией — персональные сценарии, сохранение, рекомендации, связь с библиотекой пользователя.

В MVP гибридный режим не реализуется.

## Основной источник данных

Для MVP рассматривается получение релизов через Spotify Search API с запросами вида `tag:new`.

Точные возможности `tag:new` и ограничения API должны быть проверены в ходе реализации.

## Главная ценность MVP

Дать пользователю удобный способ найти свежую музыку, которую сложно обнаружить через стандартные рекомендации Spotify.

Особенно важны сценарии:

- найти свежие релизы в конкретном жанре;
- найти свежие релизы артистов из определённой страны;
- быстро отделить более популярные релизы от менее известных;
- посмотреть только синглы или только альбомы, если API позволяет.
